## v1.2.2: Embeddingモデルの動的適用 & 解析レートリミット対策

ベクトル作成（Embedding）処理の柔軟性向上と、API利用時のレートリミット制限への耐性を強化したアップデートです。

### ⚡ Embeddingモデルの動的適用
* **モデル設定の動的反映**: これまで固定値（`text-embedding-004`）で処理されていたベクトル生成処理について、設定（Settings）画面でユーザーが指定した Embedding モデル（例: `gemini-embedding-001` 等）を動的に読み込み、処理および進捗ステータスに反映するよう改善しました。
* **統一されたモデル適用**: セマンティック（意図）検索や、不足ベクトルのバックフィル処理時にも、設定された Embedding モデルが正しく統一して適用されるように修正しました。

### 🛡️ Embedding API レートリミット対策の導入
* **待機ディレイの追加**: アプリ解析に伴う Embedding API 呼び出しの直前に `delay(800)`（0.8秒の待機時間）を挿入しました。これにより、特に一括解析（バルク）実行時に発生しやすかった API の瞬間的なリクエスト制限超過（バーストレートリミットエラー）を回避し、より安定した解析動作を実現しました。
