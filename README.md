**Insight Launcher** は、Google Gemini API を活用し、インストールされているアプリの「意味」や「用途」を自動で解析・スマートに分類する、全く新しいAndroid用ランチャーアプリです。
美しい宇宙空間のダイナミック背景（Space Background）に包まれたUIと、AIアシスタント機能があなたのスマートフォンライフをスマートに彩ります。

> [!NOTE]
> 本アプリは Google AI Studio プロジェクトとしても管理されています。
> AI Studio でアプリを表示：[AI Studio - Insight Launcher](https://ai.studio/apps/56c5aa91-770a-409c-a9d3-57814b978afd)

## 👑 アップデート情報

* **🌌 お気に入りスライド並び替え時の要素シフト＆アプリ詳細スワイプ切り替えの極上 HorizontalPager 対応 (v1.5.8) [NEW]**
  - **お気に入り並び替えを「スライド（シフト）式」に改善**: お気に入りの長押しドラッグ＆ドロップ並び替えにおいて、要素が単に入れ替わる（Swap）だけだったバグを、隣の要素をスマートに押し出して順序をシフトさせる「Remove & Add」形式に修正。通常画面、コンパクト画面、およびコンパクトダッシュボードのすべてに適用しました。
  - **アプリ詳細の HorizontalPager 移行による極上のめくり心地**: アプリ個別の詳細ダイアログ (`AppDetailsDialog`) の左右スワイプ切り替えロジックを Jetpack Compose 純正の `HorizontalPager` へ全面刷新。縦スクロール（`verticalScroll`）とのジェスチャー競合をフレームワークレベルで完全に解決し、本をめくるように吸い付く滑らかなスワイプと、フリックでどんどんめくれる高級感溢れるインタラクションを実現しました。

* **🌌 詳細カードのWeb遷移サポート＆メモリ追加・編集画面 of スクロール対応 (v1.5.7)**
  - **詳細カードからのブラウザ遷移**: AIアシスタント画面で取得・要約した GitHub リポジトリや F-Droid アプリの詳細表示において、最上部の紹介カードをタップすることで、直接ブラウザから該当リポジトリの URL や F-Droid の公式詳細ページへジャンプできるリンク機能を実装しました。
  - **メモリ（LLM Wiki）追加・編集時のUI崩れバグ解消**: 長期記憶（LLM Wiki）の追加・編集ダイアログ（`AlertDialog`）のレイアウトに縦スクロール対応（`Modifier.verticalScroll`）を適用し、ネストされていた `LazyColumn` をすっきりとした `forEach` ループに刷新。キーボード表示時にテキストフィールドやカテゴリ選択肢が潰れて視認できなくなるバグを解消し、アクセシビリティを劇的に向上させました。
  - **配色統一とテーマ連動のさらなる強化**: ホーム画面、設定画面、AIアシスタント画面の各種カードの境界線（`cardBorderColor` / `borderColor`）や区切り線、検索バーの境界線、ベクター検索のアクティブカラーにテーマのプライマリ色を動的にブレンド。テーマ切り替え時に、設定したテーマカラー（青、緑、琥珀、コズミック等）がアプリ全体に美しくシンクするよう、極上のグラスモルフィズムUIへ進化させました。

* **🌌 お気に入り長押し競合バグの完全解消＆AI検索結果のマークダウンレンダリング対応 (v1.5.6)**
  - **長押しジェスチャー競合の完全排除**: お気に入りドラッグ並び替え画面において、ドラッグ対象アイテム側の `onLongClick`（通常詳細ダイアログを開く機能）を完全に `null`（無効化）に設定。Compose の長押しイベント消費のバグを回避し、常に100%確実に長押しからスムーズにドラッグ並び替えを開始できるように改善しました。
  - **AI詳細解説のマークダウンレンダリング対応 (`MarkdownText`)**: AIアシスタント画面で取得・解析した GitHub リポジトリや F-Droid アプリの詳細解説（README要約など）を、プレーンテキスト表示からカスタム의マークダウンパーサー (`MarkdownText`) を用いたリッチレンダリング表示へと進化させました。見出し（`#`, `##`, `###`）や箇条書きリスト（`-`, `*`, `•`）の自動インデント、および `**太字**` のボールド表示に対応し、視認性を劇的に向上させました。

* **🌌 ライトテーマの廃止（ダークテーマ特化）＆コンパクト表示お気に入りドラッグバグ修正 (v1.5.5)**
  - **ライトテーマの廃止と自動ダーク移行**: 可読性低下を完全に防ぐためライトテーマを廃止（設定画面の選択肢から除外）。現在ライトテーマを設定中の環境も自動的に標準の `dark_charcoal` に移行・統一されます。
  - **ダークテーマ適用範囲の拡大（磨りガラスダイアログ）**: 設定画面の「カスタム背景URL入力」や「MCPサーバー追加」などのダイアログ（`AlertDialog`）の背景を `panelBgColor` (半透明ダーク) に変更し、背後の宇宙背景が美しく透ける高級感溢れるダークテーマデザインに統一しました。
  - **コンパクトダッシュボードでのお気に入りドラッグ修正**: コンパクト表示時の「すべて」タブ（ダッシュボード）におけるお気に入り一覧を、横スクロールのRow形式から **2列の折り返しグリッド形式 (`CompactDraggableFavoritesDashboardGrid`)** に刷新。これにより横スクロールとの競合バグを排除し、直感的な長押しドラッグ＆ドロップでの並び替えが完全にスムーズに動作するようになりました。

* **🌌 ライトテーマの背景透過度調整 & 設定画面の視認性バグ修正 (v1.5.4)**
  - **背景画像の視認性向上**: ライトテーマ選択時に宇宙背景画像の上に被せていた白オーバーレイの不透明度（透過率）を `0.85f` から `0.45f` に引き下げ、ライトテーマでも美しい背景画像がはっきりと見えるように改善しました。
  - **グラスモルフィズムによる視認性確保**: 背景画像をはっきり見せることに伴い、ホーム画面や設定画面、AIアシスタント画面の各カードの背景を、不透明度の高い白（`Color(0xDDFFFFFF)` ＝ 86% 不透明の白）に調整。これにより、美しい背景の上に白いカードが磨りガラス状に浮かび上がり、その上の黒文字が完全に可読される上質なデザインへ昇華させました。
  - **設定画面のハードコードカラー撲滅**: 設定画面（`SettingsScreen.kt`）内にハードコードされていた白文字（`Color.White`）や、ダークテーマ用の半透明白の枠線などのカラー指定を一括で動的変数に修正。ライトテーマ時の視認性を完璧に改善しました。

* **🎨 ライトテーマのテキスト視認性・コントラストの大幅向上 (v1.5.3)**
  - **コントラストの大幅強化**: ホーム画面や設定画面において、ライトテーマ時の文字色やカード背景、境界線の明暗比率（コントラスト）を調整し、明るい背景上での輪郭と文字の読みやすさを劇的に改善しました。
  - **AIアシスタント画面のライトテーマ完全対応**: 従来ダーク系配色で固定されていたAIアシスタント画面を、カラーテーマ設定に応じて配色・文字色が動的に変わるようにリファクタリング。チャットバブルやダイアログのすべてがライトテーマで美しく表示されるようになりました。

* **🛠️ コンパクトビューでのお気に入りドラッグ＆ドロップ対応 (v1.5.2)**
  - **コンパクトビュー (2列グリッド) での並び替え**: チップで `Favorites` を選択している際、またはお気に入りカテゴリ表示時において、アプリアイコンを長押ししながらドラッグすることで、自由に並び替えられるよう `CompactDraggableFavoritesGrid` を導入しました。

* **📥 AIによるWiki一括インポート（メディア/テキスト対応）＆エクスポート機能 (v1.5.1)**
  - **メディア＆ファイルからのAIナレッジ自動構築**: AIアシスタントのWiki（長期記憶）画面の上部に「アップロード（インポート）」および「ダウンロード（エクスポート）」ボタンを実装。画像、音声、動画、PDF、テキストなどの各種ファイルをアップロードするだけで、Gemini APIが内容を解析。トピックごとに自動分割・整理したWikiカードを自動で生成・登録します。
  - **既存メモリとのスマートマージ**: インポート時に既存のWikiカードとの重複や更新内容をAIが自動検知。既存のカードはIDを維持したまま最新情報へスマートに更新・マージし、不要な重複カードの発生を防ぎます。
  - **WikiのMarkdownエクスポート**: 蓄積したAIの記憶カード群を一括で整理されたMarkdownドキュメント（`ai_launcher_memory.md`）としてローカルストレージへダウンロード・エクスポートできるようになりました。

* **⭐ お気に入りドラッグ並び替え＆2列 Compact View 最適化 (v1.5.0)**
  - **お気に入りのドラッグ＆ドロップ並び替え**: コンパクトダッシュボードのお気に入り一覧に `CompactDraggableFavoritesRow` コンポーネントを導入。アプリアイコンを長押ししながらスライドするだけで、お好みの並び順に直感的に整理できるようになりました。
  - **2列の Compact ビュー最適化**: `COMPACT` ビューモード選択時のアプリアイコン一覧レイアウトを最適化。左右2列のグリッド形式（`chunked(2)`）でアイコンとAI要約文を均等に配置し、省スペースかつスマートなアライメント表示を実現しました。
  - **Firebase Applet 連携の基盤追加**: Web Applet などとのシームレスな接続用設定のサポートを強化しました（※設定ファイル等のセキュリティ対策を同時適用）。

    - ライトテーマが選択された際、宇宙背景の上に半透明の白いオーバーレイレイヤーを自動で被せ、明るい背景へと動的に変化させます。
    - 同時に、アプリアイコン名や設定カード内のテキストカラー、枠線、仕切り線、ナビゲーションドックの配色などをライトテーマ用に最適化。黒文字を基調とした高いコントラストで、文字の視認性・可読性を完璧に維持しました。

* **🌐 GitHub/F-Droid詳細情報のインテリジェントAI要約＆APK直接ダウンロード (v1.4.8)**
  - AIアシスタント画面で推薦された GitHub リポジトリや F-Droid アプリについて、さらに詳細な情報を対話形式で取得・表示できるようになりました。
  - **GitHubリポジトリ要約**: 推奨されたGitHubリポジトリをタップした際、そのプロジェクトのREADME（`README.md`/`readme.md`）をバックグラウンドで自動取得。Geminiがコード構造、主な用途、インストール・開始方法などを、ユーザーが設定した言語（日本語・英語・韓国語・中国語など）に合わせて美しくMarkdownで要約し、アプリ内で即座に確認できます。
  - **F-Droidアプリ要約＆APK抽出**: F-Droidのパッケージを検索・推薦された際、F-Droidのウェブサイトを自動でスクレイピングして詳細情報を抽出。Geminiによる分かりやすい日本語要約（ライセンス、主要機能、ターゲット等）に加え、F-Droidのページから**直接APKファイルのダウンロードリンク（上位最大5件）**を自動抽出し、ワンタップで手軽にダウンロードできるダイレクトインストール連携を実装しました。

* **📱 超高密度「コンパクト」レイアウトモードの追加と詳細UIの洗練 (v1.4.7)**
  - ホーム画面のアプリ一覧表示形式に、第3の選択肢として情報密度を極限まで高めた「`COMPACT` (コンパクト)」表示モードを追加しました。
  - コンパクトモードでは、32.dpのミニマムアイコンによる配置と、アイコン＋1行AI要約（`summary`）をスマートに一覧化するハイブリッドリスト表示がサポートされ、一覧性と情報量が両立した効率的なビューを選択できるようになりました。
  - アプリ詳細ダイアログのUIを拡張。パッケージ名の下に、AIが自動分類したカテゴリ名（`category`）を視認性の高い青色のボーダー付きデザインバッジで常時表示する機構を追加しました。
  - アプリ詳細ダイアログの「カスタム絵文字アイコン」Preset一覧部分を `LazyRow` スクロール形式へ改善し、横幅の狭い端末でもボタンが折り返されず綺麗にスライド選択できるよう最適化しました。

* **🌐 Google Search Grounding による実在ストアアプリのリアルタイム検索・推薦 (v1.4.6)**
  - AIアシスタントが未インストールのおすすめストアアプリを提案する際、Gemini純正の「Google検索グラウンディング (googleSearchRetrieval)」ツールを統合してリアルタイムにWeb検索を実行するアップデートを行いました。
  - 従来の不安定なPlayストアHTMLスクレイピング（正規表現パース）を排除し、AIがライブ検索から正確な実在アプリ名、正しいパッケージ名（`com.instagram.android` 等）、および正規のPlayストアURLを完全に保証して提案できるようになりました。
  - 検索グラウンディングとFunction Calling（関数呼び出し）が競合してレスポンスJSON構造が損なわれるのを防ぐため、`generationConfig` のスキーマ適用ロジックをリファクタリングし、構造化出力の整合性を完璧に維持しました。
  - アバター画像等の非同期読み込み負荷を低減し、スクロール時のパフォーマンス向上を目的としたメモリ最適化（プレースホルダーへの整理）をあわせて実施しました。

* **🌐 GitHub連携カードのUI刷新とAI深掘り詳細検索 (v1.4.5)**
  - AIアシスタント内におけるGitHub検索結果のリポジトリカードのUIデザインを大幅に強化しました。
  - 各リポジトリカードの右下に「詳細検索（虫眼鏡）」ボタンを追加。タップするだけで、「リポジトリ『owner/repo』のさらに詳細な情報を教えてください」という対話クエリが自動送信され、シームレスにAIに対象プロジェクトのコード構造やREADME解説を深掘り質問できるようになりました。
  - ブラウザ遷移（`ACTION_VIEW`）時のシステム例外処理を `try-catch` で多重ガードし、ブラウザ未設定などの極めて稀なケースでもアプリが強制終了しない安全設計を導入しました。

* **🧠 長期記憶（LLM Wiki）のマインドマッピング＆自律的ナレッジグラフ化 (v1.4.4)**
  - AIアシスタントが会話から記憶すべき内容を自動判別し、自律的にナレッジカードの作成や編集を提案・プレビュー提示する機能を追加しました。
  - 記憶エンティティに「`tags` (タグ)」および「`relatedLinkIds` (関連リンク)」の概念を拡張。Roomデータベースをバージョン4へバンプ（JSONシリアライズ用の `Converters.kt` を新規導入）し、記憶カード同士を双方向のチェックボックス操作で相互接続するマインドマップ型のUIを設定・編集画面に実装しました。
  - 記憶登録時に5つ以上のタグ付与を必須化し、AIがコンテキストを読み取る際の検索精度と関連知識の紐付け能力を劇的に向上させました。

* **🌓 背景輝度解析の初期化タイミングバグの修正 (v1.4.3)**
  - Android起動時やViewModelのロード時に、宇宙背景画像の明暗を解析してアプリアイコン名などのテキストコントラスト（白文字/黒文字の動的切替）を自動調整する機能（Luminance Analysis）が、まれにバインディング順序の不整合によって動作しない不具合を修正しました。
  - `_bgLuminance` と `autoContrast` の定義・宣言位置をViewModel内の初期化部上部に整理し、Kotlinの初期化フェーズと完全に同期させることで、背景画像選択時や起動直後の文字色自動コントラスト調整の動作安定性を劇的に高めました。

* **📚 GitHub検索結果リポジトリのリアルタイム自動翻訳 (v1.4.2)**
  - AIアシスタントを通じてGitHub上のオープンソースプロジェクトやコードテンプレートを検索する際、GitHub APIから取得される元の英語の紹介文（`description`）を、Geminiを用いて自動的かつリアルタイムでスマホの表示言語（日本語・韓国語・中国語など）に一括翻訳（`translateGitHubRepos`）する機能を追加しました。
  - 検索結果の上位10件のリポジトリ説明文が一括で自然なローカライズ翻訳へ差し替えられてチャット画面のカードに表示されるため、英語圏のリポジトリ情報も言語の壁を感じずに瞬時に把握できるようになりました。

## ✨ 主な機能

* **🌌 フローティングナビゲーションドック (Floating Navigation Dock) [NEW]**
  - **常時アクセス可能なUIドック**: ホーム画面下部に、美しく浮かび上がる半透明なクイックナビゲーションドックを搭載。
  - **3つのクイックアクション**: 「AIアシスタント画面」「表示レイアウト（グリッド/リスト）切り替え」「設定画面」へ、どの画面からでも即座に遷移・操作が可能です。
  - **重なり防止とインセット最適化**: Androidのナビゲーションバー部分との重なりを防止するパディング処理や、リスト表示などの背後コンテンツとの適切な余白処理（130.dp）を自動で適用します。

* **⚡ 高速一括AI解析機能 (Bulk Analysis) & 設定の改善**
  - **選べる解析モード**: 未解析の全アプリを解析する際、1つのAPIリクエストで複数アプリをまとめて高速・低コストで処理する「高速一括解析 (バルク)」と、1アプリずつ個別に解析する「丁寧な順次解析 (個別)」の2モードから選択可能。
  - **解析ステータスのリアルタイム可視化**: 解析実行中、使用されているGeminiモデル名（プライマリかバックアップか）やベクトル生成（Embedding）フェーズであることをステータスエリアに詳細表示。最大4行まで折り返し可能に。
  - **主要モデルの簡易切り替え**: 設定画面にプライマリおよびバックアップモデル（`gemini-flash-latest` / `gemini-flash-lite-latest`）のクイック選択ボタンを追加し、ワンタップで切り替え可能。
  - **Gemmaモデル自動判別**: ツール呼び出し（Function Calling）や構造化出力に対応していない Gemma モデルでの動作を最適化し、安定したAI動作を実現。
  - **丁寧なRate Limitハンドリング**: API利用回数制限（HTTP 429）時のエラーメッセージを刷新。具体的な原因や、マイAPIキー設定などの解決方法をわかりやすく案内します。

* **🤖 Model Context Protocol (MCP) 連携機能の搭載**
  - Gemini APIのFunction Calling機能と統合されたMCPクライアントエンジンを搭載。
  - **ビルトインツール一覧**:
    - `get_current_time_and_date`: 正確なシステム日時・タイムゾーン・曜日の取得。
    - `get_device_status`: バッテリー残量、ストレージ残量、RAM使用量、ネットワーク状態（WIFI/モバイルなど）の取得。
    - `search_installed_apps`: キーワードによるインストール済みアプリの検索。
    - `launch_installed_app`: 指定されたパッケージ名によるアプリの直接起動。
    - `launcher_settings_control`: 背景画像（URL）、表示レイアウト（グリッド/リスト）、アイコン形状、AI応答言語、自動コントラスト切り替えなどのランチャー設定の読み書き。
    - `evaluate_math_expression`: 四則演算、平方根、三角関数などの数式の正確な評価。
    - `get_weather_info`: Open-Meteo APIを用いた指定都市のリアルタイム天気情報の取得。
  - **カスタムMCPサーバー**:
    - JSON-RPCベースの外部MCPサーバーを追加して、独自ツール（ファイル操作、ローカルスクリプト等）をAIアシスタントに実行させることが可能。
    - 各サーバーのトグルスイッチによる有効/無効の切り替えや、削除にも対応。

* **⚡ アプリアイコンキャッシュ＆パフォーマンス最適化**
  - アプリアイコンのロード処理にキャッシュ機構（`AppIconCache` / `SharedPreferencesManagerCache`）を導入し、バックグラウンドでの非同期ロードおよびデバイス解像度に合わせた適切なリサイズ処理を実現。
  - アプリ一覧スクロール時におけるメインスレッドの負荷を劇的に減らし、非常になめらかでカクつきのない高速なスクロール動作を提供します。
* **🔄 ホーム画面復帰時のアプリ同期自動化 [NEW]**

## 🛠️ ローカルでの開発 & ビルド方法

### 前提条件
* **Android Studio** (最新版推奨)
* **JDK 17** (ビルドに必要です。`mise` 等での管理を推奨)
* **Android SDK** (API 24以上、Target API 36)

### セットアップ手順

1. **プロジェクトのインポート**
   Android Studio を開き、本プロジェクトのルートディレクトリをインポートします。

2. **APIキーの設定 (`.env` ファイルの作成)**
   プロジェクトのルートディレクトリに `.env` ファイルを作成し、Gemini APIキーを設定してください。
   ```bash
   cp .env.example .env
   ```
   `.env` ファイルを開き、以下のようにキーを入力します：
   ```env
   GEMINI_API_KEY=YOUR_GEMINI_API_KEY_HERE
   ```

3. **SDKパスの確認 (`local.properties`)**
   `local.properties` を作成または編集し、お使いの環境の Android SDK のパスを指定してください。
   ```properties
   sdk.dir=/path/to/your/Android/Sdk
   ```

4. **ビルド & 実行**
   ```bash
   # デバッグビルド (APKの生成)
   ./gradlew assembleDebug
   ```
